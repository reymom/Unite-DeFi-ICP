# Full flow for a Swap between ICP and EVM

For these tests we will need to set up the relayer and orderbook as described in [2](2.%20relayer-auction-test.md).

We will also need to deploy the factory, setting up the escrow wasm, and deploy the ledger canister. This is described in [4](4.%20escrow-factory-test.md).

We will act as a single resolver for this example, but we will use the whole architecture that allows a fully-fledged 1inch fusion+ swap between ICP and Ethereum.

Deploy:

1. The orderbook: `dfx deploy orderbook`.
2. The factory: `dfx deploy factory --with-cycles 1_000_000_000_000 && dfx canister call factory set_escrow_wasm --argument-file tmp/escrow_arg_hex.did`.
3. ICP ledger (see [4](4.%20escrow-factory-test.md)):

```bash
dfx identity use taker
export TAKER_ACCOUNT_ID=$(dfx ledger account-id)
export TAKER_PRINCIPAL=$(dfx identity get-principal)

dfx identity use maker
export MAKER_ACCOUNT_ID=$(dfx ledger account-id)
export MAKER_PRINCIPAL=$(dfx identity get-principal)

dfx identity use default
export DEFAULT_ACCOUNT_ID=$(dfx ledger account-id)
export DEFAULT_PRINCIPAL=$(dfx identity get-principal)

dfx deploy icp_ledger_canister --argument "
  (variant {
    Init = record {
      minting_account = \"$DEFAULT_ACCOUNT_ID\";
      initial_values = vec {
        record {
          \"$TAKER_ACCOUNT_ID\";
          record {
            e8s = 10_000_000_000 : nat64;
          };
        };
        record {
          \"$MAKER_ACCOUNT_ID\";
          record {
            e8s = 15_000_000_000 : nat64;
          };
        };
      };
      send_whitelist = vec {};
      transfer_fee = opt record {
        e8s = 10_000 : nat64;
      };
      token_symbol = opt \"ICP\";
      token_name = opt \"Local ICP\";
    }
  })
"
```

4. The relayer: `dfx deploy relayer --argument-file tmp/relayer_init.did`.
5. EVM RPC: `dfx deps pull && dfx deps init && dfx deps deploy evm_rpc`.
6. The resolver: `dfx deploy resolver --argument-file tmp/resolver_init.did`

## Order Intent

As maker, I want to swap `100 ICP` for at least `2300 1INCH` on `Arbitrum`. We need to signal the order intent to the relayer. We will do this later in the frontend, but we can also call the relayer directly and skip the UX proxy. We will:

1. Approve the factory to spend 100 ICP in our behalf.

```bash
dfx identity use maker
dfx canister call icp_ledger_canister icrc2_approve '(record {
  from_subaccount = null;
  spender = record { owner = principal "'$(dfx canister id factory)'"; subaccount = null };
  amount  = 100_000_000_000 : nat;
  expected_allowance = null;
  expires_at = null;
  fee = null;
  memo = null;
  created_at_time = null;
})'
```

2. Register the order intent in the orderbook via relayer.

```bash
set -euo pipefail

FRIENDLY="my friendly secret phrase"
SECRET_HEX=$(printf '%s' "$FRIENDLY" | openssl dgst -sha3-256 -binary | xxd -p -c 256)
HASHLOCK_HEX=$(printf '%s' "$SECRET_HEX" | xxd -r -p | openssl dgst -sha3-256 -binary | xxd -p -c 256)
secret_vec=$(python3 - <<PY
h = bytes.fromhex("$SECRET_HEX")
print("; ".join(str(b) for b in h))
PY
)

hashlock_vec=$(python3 - <<PY
h = bytes.fromhex("$HASHLOCK_HEX")
print("; ".join(str(b) for b in h))
PY
)

NOW=$(date +%s)

cat > tmp/submit_order_icp.did <<EOF
(
  record {
    order_hash = "0xaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaa";
    signature = blob "\01\02\03\04";
    maker_asset = variant { ICP };
    taker_asset = variant {
      Erc20 = record {
        chain_id = 1 : nat64;
        address = opt "0x1111111111111111111111111111111111111111";
      }
    };
    maker_icp = principal "$MAKER_PRINCIPAL";
    maker_evm = "0xC87466D98c7AF57A02F7c451360C1Ac3529A99E6";
    making_amount = 4_500_000_000 : nat;
    salt = "some-salt-1234";
    maker_traits = opt "limit";
    auction_start_at = $(($NOW - 10)) : nat64;
    waiting_period = 300 : nat64;
    auction_start_rate = 1.25 : float64;
    min_return_amount = 5_343_750_000 : nat64;
    price_curve = record {
      points = vec {
        record { time_offset_secs = 0   : nat64; price_multiplier = 1.20 : float64 };
        record { time_offset_secs = 300 : nat64; price_multiplier = 1.05 : float64 };
        record { time_offset_secs = 600 : nat64; price_multiplier = 0.90 : float64 };
        record { time_offset_secs = 900 : nat64; price_multiplier = 0.80 : float64 };
      }
    };
    safety_deposit = 500_000_000 : nat64;
    timelocks = record {
      finality_lock = 60 : nat64;
      withdrawal = 300 : nat64;
      public_withdrawal = 600 : nat64;
      cancellation = 900 : nat64;
      public_cancellation = opt 1200;
    };
    hashlock = vec { $hashlock_vec };
  },
  vec { $secret_vec }
)
EOF
```

```bash
dfx identity use maker && dfx canister call relayer submit_order --argument-file tmp/submit_order_icp.did
```

## Resolver

The relayer will immediately discover the order and will initiate the dutch auction with the parameters specified in the order, the deployed resolver will

```bash
dfx identity use taker
dfx canister call relayer accept_price '("0xaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaa", "0xC87466D98c7AF57A02F7c451360C1Ac3529A99E6")'
```

We need to create the escrow in ICP's leg:

```bash
cat > tmp/create_icp_source_escrow.did <<EOF
(
  record {
    role         = variant { Source };
    initiator    = "$MAKER_PRINCIPAL";
    counterparty = "$TAKER_PRINCIPAL";
    asset        = variant { ICP };
    amount            = 4_500_000_000 : nat64;
    hashlock          = vec { $hashlock_vec };
    timelock          = record {
      finality_lock = 60 : nat64;
      withdrawal = 300 : nat64;
      public_withdrawal = 600 : nat64;
      cancellation = 900 : nat64;
      public_cancellation = opt 1200;
    };
    payout_addr    = "$TAKER_PRINCIPAL";
    safety_deposit = 500_000_000 : nat64;
  }
)
EOF
```

```bash
dfx canister call --wallet $(dfx identity get-wallet) --with-cycles 800000000000 factory create_escrow --argument-file tmp/create_icp_source_escrow.did
```

We can check is deployed and funded on the ICP leg. Take the id of the new deployed escrow given in the last output:

```bash
dfx canister call icp_ledger_canister icrc1_balance_of \
  '(record { owner = principal "ucwa4-rx777-77774-qaada-cai"; subaccount = null })'
```

We will see the outpu `(5_000_000_000 : nat)`.

## Verify

We can verify the ICP leg (src):

```bash
# get the escrow principal id first if you failed to capture the call output from the factory
dfx canister call factory list_escrows '()'
```

On the evm leg, we can use the `rpc_evm` linked to the resolver to manually deploy the escrow.

```bash
dfx canister call resolver deploy_evm_escrow '(
  "0xaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaa"
)'
```

## Withdraw funds

Once deployed source and destination escrows, we can signal the relayer that we deployed properly. It will verify and return the secret.

```bash
ORDER_HASH="0xaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaa"
RESOLVER_EVM="0xC87466D98c7AF57A02F7c451360C1Ac3529A99E6"
EVM_ESCROW="0xdeadbeefdeadbeefdeadbeefdeadbeefdeadbeef"
ICP_ESCROW="vizcg-th777-77774-qaaea-cai"

dfx canister call relayer verify_and_reveal_secret '(
  "'"$EVM_ESCROW"'",
  "'"$ICP_ESCROW"'",
  "'"$ORDER_HASH"'",
  "'"$RESOLVER_EVM"'"
)'
```

With the secret, we can proceed to withdraw both legs escrows funds

On the icp leg:

```bash
SECRET_BLOB="\11\d3\0a\96\11\a9\5b\59\21\17\05\03\ff\65\bd\e4\e0\68\40\a6\bd\74\b3\1d\c8\55\c8\2e\f8\c2\45\7e"

dfx canister call $ICP_ESCROW withdraw "(blob \"$SECRET_BLOB\")"
```

On the evm leg:

```bash
dfx canister call resolver withdraw_evm_manual "(
  \"$ORDER_HASH\",
  blob \"$SECRET_BLOB\"
)"

```
