# Local factory test: ICP escrow (native + ICRC‑2 token)

## 1. Build & deploy canisters

```bash
# build escrow + factory WASM
cargo build -p escrow  --release --target wasm32-unknown-unknown
cargo build -p factory --release --target wasm32-unknown-unknown

# update .did files
candid-extractor target/wasm32-unknown-unknown/release/factory.wasm > backend/factory/factory.did
candid-extractor target/wasm32-unknown-unknown/release/escrow.wasm  > backend/escrow/escrow.did

# deploy factory with plenty of cycles
dfx deploy factory --with-cycles 1_000_000_000_000
```

### (one‑time) upload escrow WASM to factory

Generate the escrow wasm argument as a hex-escaped did in tmp/escrow_arg_hex.did:

```bash
python3 scripts/gen_escrow_arg_did.py
```

And set the escrow:

```bash
dfx canister call factory set_escrow_wasm --argument-file tmp/escrow_arg_hex.did
```

## 5. Taker creates ICP destination escrow via factory

```bash
SECRET='some‑secret'
HEX=$(echo -n "$SECRET" | python3 -c 'import sys,hashlib; print(hashlib.sha3_256(sys.stdin.buffer.read()).hexdigest())')
RAW=$(echo "$HEX" | sed 's/../\\&/g')
````

```bash
cat > tmp/create_escrow.did <<EOF
(
  record {
    role         = variant { Destination };
    initiator    = "$MAKER_PRINCIPAL";
    counterparty = "$TAKER_PRINCIPAL";
    asset        = variant { ICP };
    amount            = 4_500_000_000 : nat64;
    hashlock          = blob "$RAW";
    timelock          = record {
        finality_lock       = 20  : nat64;
        withdrawal          = 40  : nat64;
        public_withdrawal   = 60  : nat64;
        cancellation        = 80  : nat64;
        public_cancellation = opt 100;
    };
    payout_addr    = "$MAKER_PRINCIPAL";
    safety_deposit = 500_000_000 : nat64;
  }
)
EOF
```

```bash
dfx canister call --wallet $(dfx identity get-wallet) \
  --with-cycles 800000000000 \
  factory create_escrow \
  --argument-file tmp/create_escrow.did
```

```bash
dfx canister call --wallet $(dfx identity get-wallet) --with-cycles 800000000000 factory create_escrow --argument-file tmp/create_icp_source_escrow.did
```

Verify the Factory Canister's Balance:

### verify deployed escrow

```bash
dfx canister call ulvla-h7777-77774-qaacq-cai get_escrow '()' --candid backend/escrow/escrow.did
```

> Note: the escrow canister was created dynamically by the factory and we're calling it by its raw principal, DFX doesn’t have a local "name → candid interface" mapping, so it can't auto-resolve the human-readable types and falls back to inferred (numbered) encoding if we don't provide the did.

If you missed the output when deploying the escrow from the factory canister, you can always check the principal of the escrow canister from the list of escrows registered in the factory:

```bash
dfx canister call factory list_escrows '()'
(
  vec {
    record {
      canister_id = principal "ulvla-h7777-77774-qaacq-cai";
      state = variant { Open };
      params = record { ... };
    };
  };
)
```

## 6. Maker withdraws with the secret

```bash
SECRET_HEX="$(printf 1 | xxd -p -c 32)"
dfx identity use maker
dfx canister call "$ESCROW_ID" withdraw "(blob \"0x$SECRET_HEX\")"
```

_Escrow pays amount to maker’s account and safety_deposit to maker (the executor)._

## 7. Query final state

```bash
dfx canister call "$ESCROW_ID" get_escrow
```

Expected: `status = variant { Completed }`

## Manual funding path (ICRC‑1 only)

1. Deploy another escrow with a token that does not implement the standard icrc-2.
2. Taker icrc1_transfers tokens to the escrow principal.
3. Taker calls lock() (no parameters).
4. Flow continues as above.
